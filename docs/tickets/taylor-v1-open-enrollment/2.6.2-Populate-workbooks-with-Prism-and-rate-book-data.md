# 2.6.2 Populate workbooks with Prism and rate book data

## Traceability
- **Asana task ID:** `1213863006807798`
- **Section:** Backlog

## Product intent

Fill renewal workbooks with **Prism-sourced** plan/census/enrollment data and **rate-book / auxiliary** data (carrier rate sheets, zip or regional files) so outputs match operational quality today—previously done in large part via **BabyTaylor** (Prism API calls, new sheets, validation against external spreadsheets).

**April 2026 (Jagger / Anna):** Baby Taylor kept **rate books and rebanding files local** and primarily **verified** ClientSpace output against those files. If carrier rates land correctly in **Prism** (e.g. **future rate band**), the product may **not** need the same verification loop—confirm when implementing ([AnnaSync-WorkflowReview-4.7.26.md](../../meetings/AnnaSync-WorkflowReview-4.7.26.md), [2.5.3](./2.5.3-Rate-book-management.md)).

## Product requirements (verifiable)

- Workbook cells/ranges populated from normalized Prism and rate inputs per **approved mapping**: Prism canonical model → `CSExport`, `ClassesPlans`, `CurrentPlans`, `RenewalPlans`, class tabs (`Class 1`–`Class 7` or renamed equivalents), optional `Census`, and any other template-defined input ranges ([2.4.1](./2.4.1-Automated-ingestion-of-client-plan-census-enrollment-data-from-prism-and-rate-data-from-admin-console-upload.md), [2.2.5](./2.2.5-Adapter-mapping-layer-to-manage-Prism-schema-dependencies.md)). The mapping matrix is maintained with **Questco**; see [Renewal-Workbook-Questco-Discovery.md](../../prd/Renewal-Workbook-Questco-Discovery.md).
- After writes, the pipeline triggers a **full recalc** compatible with the template (Excel or an equivalent calc engine) so formulas, defined names, and lookups match what reps see in Excel.
- Validation hooks for mismatches (rates, regions) with clear operator or log outcomes.
- Errors, edge cases, and operator-visible states handled or explicitly deferred with tracking.

## Success criteria

- [ ] Demonstrable in non-production or staging for a representative renewal scenario.
- [ ] Verification evidence (automated tests, scripted manual test, or signed QA checklist).
- [ ] Integration touchpoints exercised with logs or captured responses suitable for audit where applicable.
- [ ] Population aligns with **Questco-approved** cell/range mapping for the template version in use.
