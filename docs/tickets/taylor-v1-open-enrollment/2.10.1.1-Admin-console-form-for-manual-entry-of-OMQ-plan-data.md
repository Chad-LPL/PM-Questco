## Product intent

Provide a **structured place** for operators to enter OMQ plan details when not sourced from Prism. **V1 direction (April 2026):** the primary capture path may be a **Questco Excel template** keyed by temps, with Taylor **ingesting that file**—see [2.5.4](./2.5.4-OMQ-plan-data-entry-and-management.md). An **admin console form** remains a valid implementation if architecture keeps all operator entry in Taylor; **ClientSpace BSR** OMQ fields are **tracking/status only**, not full quote storage (**April 8**)—revival as checklists is still compatible with template-first capture ([2.2.2](./2.2.2-ClientSpace-API-integration-and-workflow-status-updates.md)). **Product:** no strong preference between ClientSpace vs admin **if** executive reporting stays a **single system of record** ([2.3.6](./2.3.6-Admin-console-providing-renewal-status-visibility-and-light-reporting.md)).
