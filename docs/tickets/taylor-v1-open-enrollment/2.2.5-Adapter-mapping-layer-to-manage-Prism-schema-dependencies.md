- **Asana task ID:** `1213834655988117`
- **Section:** Backlog

## Product intent

Single **Prism → canonical DTO** layer so workbook and [2.4.3](./2.4.3-Data-normalization-prior-to-workbook-population.md) avoid raw JSON. Limits breakage when Prism changes ([2.2.1](./2.2.1-Prism-HR-API-Setup-authentication-connectivity-normalization.md)).

**Second deliverable:** a **workbook column / range catalog** (or mapping matrix): canonical fields and Prism fields → cells on `CSExport`, `ClassesPlans`, `CurrentPlans`, `RenewalPlans`, class tabs, `Census`, etc. Maintained with **Questco** alongside the DTO; see [Renewal-Workbook-Template-Sheet-Reference.md](../../Renewal-Workbook-Template-Sheet-Reference.md) and [Renewal-Workbook-Questco-Discovery.md](../../prd/Renewal-Workbook-Questco-Discovery.md). Feeds [2.6.2](./2.6.2-Populate-workbooks-with-Prism-and-rate-book-data.md).

### April 8, 2026 — sources beyond raw Prism fields

- **Benefit groups / classes:** Use Prism **benefit groups** (group name + ID) and **GetEmployee** group membership—not only inference from employees alone.
- **Apply plan groups:** When **OBP** is not used to hydrate ClientSpace, do **not** read ClientSpace “Apply Plan Groups.” Taylor must own a **carrier → full plan catalog** map (or config) and combine it with **rebanding** (which carrier bands exist for the client → which plan universes to show). See [2.4.1](./2.4.1-Automated-ingestion-of-client-plan-census-enrollment-data-from-prism-and-rate-data-from-admin-console-upload.md), [2.4.2](./2.4.2-Pre-generation-validation-of-plans-plan-groups-and-rate-band-designation.md), [2.4.3](./2.4.3-Data-normalization-prior-to-workbook-population.md).
- **Rates vs bands:** **Future rates** may be read from Prism after Questco import; **band assignment** is resolved via **SharePoint rebanding file** ingest ([2.2.3](./2.2.3-SharePoint-file-storage-configuration.md)).

### April 10, 2026 — Prism **import file** mapping (outbound)

- **Separate from** inbound Prism API DTOs: Taylor must emit **plan** and **benefit rules** CSV templates ([2.8.1](./2.8.1-Generate-plan-setup-and-contribution-import-files.md)) with **~full** column sets—Prism rejects incomplete headers.
- **Gap analysis pending Questco:** Not every **rules** column maps to a workbook cell (e.g., eligibility status / days coding discussed with **Sue** / **LaWanda**). Adapter owns **derivation rules** once the **spreadsheet mapping** arrives (**Anna**, **Dimple**, **Jagger** + analysts). May require **new workbook fields** ([2.6.1](./2.6.1-Generate-renewal-workbooks-using-Questco-templates.md)).
- **Edge-case fixtures:** Disability/life **tier** lines, **multi-company ID** rows—encode in tests when samples exist.

## Product requirements (verifiable)

- Documented mapping; tolerate additive Prism fields. Log drift (missing/wrong shape) for 2.2.1 errors.
- Short note: what Taylor **stores** vs **passes through**—not a full Prism replica.
- **Workbook mapping:** documented Prism (and rate upload) → template cell/range mapping for the supported template version(s), co-owned with Questco.
- Fixtures + tests; optional non-prod smoke. Downstream uses canonical types at boundaries.

- [ ] Workbook path (or stub) uses adapter only.
