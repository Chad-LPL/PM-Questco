## Product intent

Generate **Prism import files** from Taylor so analysts can review and upload them. This supports Taylor V1 open enrollment orchestration. Outputs feed the **manual review checkpoint** before production import ([2.8.3](./2.8.3-Manual-review-checkpoint-prior-to-Prism-submission.md)); include the **analyst** stakeholder in import design sessions per product owner ([AnnaSync-WorkflowReview-4.7.26.md](../../meetings/AnnaSync-WorkflowReview-4.7.26.md)).

### April 10, 2026 — two files, plan before rules

- **Two separate CSV artifacts:** **Client plan** (plan setup) **first**, then **benefit rules** (contributions, waiting periods, tier rows, eligibility coding, etc.). Prism requires **plans to exist** before rules attach. Analysts run **two separate** Prism import actions—they **cannot** upload both files in one combined step.
- **Benefit rules template** is on the order of **~82 columns**; Prism expects the **full header row** (missing columns → import failure). Prism validates **headers** and **some** required fields; **optional** columns are not clearly marked in Prism help—coordinate with **Sue** / **LaWanda** on **required-field** reality.
- **Row semantics:** **EE / ES / EC / FAM** ordering matters; **contribution** method **F** (fixed) vs **P** (percent)—**both** can appear for one benefit in some cases; certain **disability/life** patterns use **placeholder lines** so eligibility tiers still work. Generation must match **analyst-reviewed** examples.
- **Multi-entity:** One file may contain rows for **multiple company IDs** (affiliated entities sharing benefits).
- **v1 posture:** Emit **CSV** (preferred type per workshop) for **human review** and possible hand-edit before Prism; **API** direct write is a longer-term alternative ([PRD §5.9](../../prd/Taylor-v1-Open-Enrollment-Orchestration-PRD.md#59-prism-import-files--manual-checkpoint)).
- **Dependency — Questco mapping deliverable:** Column-by-column map from Prism templates → **workbook** cells, **Prism** API fields, or **derived** defaults; list of fields **not** on the workbook today (may drive **workbook template** changes). Tracked in [PRD §10](../../prd/Taylor-v1-Open-Enrollment-Orchestration-PRD.md#10-dependencies-on-questco). **Blocks** fine-grained acceptance criteria for every column.
