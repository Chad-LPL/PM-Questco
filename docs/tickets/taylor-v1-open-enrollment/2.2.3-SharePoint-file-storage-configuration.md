## Product intent

SharePoint set up for Taylor to **upload** workbooks and return **links** for [2.6.5](./2.6.5-SharePoint-storage-and-posting-URL-to-ClientSpace.md). Triggers live under [2.6.6](./2.6.6-Workbook-completion-trigger-automation.md). Today many files sit in ClientSpace; this ticket is enablement for generate → SharePoint → URL in ClientSpace.

Because Taylor runs in **Azure**, Questco expects **Azure app registration(s)** for Taylor services and **Microsoft Graph** (or equivalent app-only patterns) to authorize access to SharePoint—avoids ad-hoc key handoffs. Human access to SharePoint also follows Questco identity (Questco accounts).

Some operational spreadsheets (e.g. **broker-type** source-of-truth discussions) may live in SharePoint today; data-governance follow-ups may tie to [2.5.4](./2.5.4-OMQ-plan-data-entry-and-management.md) / product discovery, not this ticket’s core scope.

### Rebanding file (April 8, 2026)

Questco aligned on hosting the **rebanding assignment file** (all clients, updated when bands change) in **SharePoint** so Taylor can read it without a Prism-import bottleneck when bands shift late. This ticket should include a **documented folder/library**, permissions for operations and Taylor’s app identity, and how **updates/replacements** are versioned or audited. Consumption logic is tracked in [2.4.1](./2.4.1-Automated-ingestion-of-client-plan-census-enrollment-data-from-prism-and-rate-data-from-admin-console-upload.md) and [2.5.3](./2.5.3-Rate-book-management.md).

### Executed workbook & benefit-guide assets (April 9, 2026)

- **Inbound / staging:** Architecture may use a **global** or **upload-only** library/folder for renewal reps, with Taylor **moving** or **routing** files using **client ID in the workbook**—reduces wrong-folder mistakes; permissions model should be explicit ([2.6.6](./2.6.6-Workbook-completion-trigger-automation.md)).
- **Static document library:** **Benefit guide** boilerplate pages (annual IRS updates, carrier-static pages) may live in SharePoint for Taylor to assemble with dynamic M/D/V pages ([2.7.1.3](./2.7.1.3-Automated-generation-of-the-Benefit-Guide.md)).
