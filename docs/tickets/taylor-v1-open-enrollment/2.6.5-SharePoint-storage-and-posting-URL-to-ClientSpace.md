## Product intent

**SharePoint** is the durable store for Taylor-generated **renewal** workbooks and related artifacts; **ClientSpace** holds operational context—URLs back to SharePoint keep humans in one workflow.

### Generated renewal workbook (existing scope)

- After generation, Taylor **stores** the file in SharePoint and **writes the URL** to the appropriate ClientSpace field / batch so reps open the latest workbook from ClientSpace.

### Executed workbook (April 9, 2026)

- When the **executed** workbook lands in Taylor’s pipeline (however triggered—[2.6.6](./2.6.6-Workbook-completion-trigger-automation.md)), Taylor should **persist** it in SharePoint (or move from a **staging** folder) and **post the SharePoint link** to ClientSpace—**reps should not** manually paste a second link if Taylor can set it. Aligns with [2.2.2](./2.2.2-ClientSpace-API-integration-and-workflow-status-updates.md).
