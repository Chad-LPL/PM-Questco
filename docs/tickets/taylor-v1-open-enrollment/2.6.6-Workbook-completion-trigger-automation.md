## Product intent

Automation that **starts Taylor’s pipeline** when a **completed or executed** workbook is ready for ingestion—today described as **SharePoint-triggered events**, with **additional patterns** validated in discovery.

### April 9, 2026 — trigger options (executed workbook path)

- **Problem:** ClientSpace does not offer a **reliable webhook** pattern for “file arrived,” so the team explored **SharePoint** listeners, a **single global upload folder** where Taylor **routes files to the correct client** using **client ID embedded in the workbook** (avoids misfiled per-client folders), and **ClientSpace data forms / HTTP** calling an endpoint Taylor hosts when a workbook is uploaded.
- **Rep workflow:** Minimize duplicate steps: if the rep uploads to SharePoint (or ClientSpace forwards the file), **Taylor** should **post the SharePoint URL** into ClientSpace ([2.6.5](./2.6.5-SharePoint-storage-and-posting-URL-to-ClientSpace.md)) rather than requiring a second manual link.
- **Permissions:** Consider **upload-only** access to a **staging / global** folder for renewal reps vs broader access to Taylor-managed destination folders (reduces accidental delete/misfiling).

Coordinate with [2.2.3](./2.2.3-SharePoint-file-storage-configuration.md), [2.3.4](./2.3.4-Submission-process-for-completed-workbook-ingestion.md), [2.6.5](./2.6.5-SharePoint-storage-and-posting-URL-to-ClientSpace.md), [2.6.7](./2.6.7-Retrieval-parsing-normalization-and-validation-of-executed-workbooks.md).
