- **Asana task ID:** `1213866553394542`
- **Section:** Backlog

## Product intent

Validate **plans**, **plan groups**, and **rate band** designations **before** workbook/document generation so Taylor fails fast with actionable errors instead of broken outputs. Aligns with operational pain from **OBP import**: Prism and ClientSpace must agree on **rate group titles**, **plan IDs**, and **benefit group codes**—misalignment blocks import and shows as errors until corrected in ClientSpace or Prism ([UserStoryDiscovery-DemoFlow-4.1.26.md](../../meetings/UserStoryDiscovery-DemoFlow-4.1.26.md)). As the renewal spine moves toward **Prism API** pulls instead of ClientSpace OBP import ([2.3.1](./2.3.1-Renewal-state-machine-with-defined-milestones-and-transitions.md)), validation rules must still reflect **dual-system consistency** wherever both stores are in play during migration ([AnnaSync-WorkflowReview-4.7.26.md](../../meetings/AnnaSync-WorkflowReview-4.7.26.md)).

### Discovery notes — April 8, 2026

- **Apply plan groups / full carrier catalog:** When OBP-style ClientSpace sync is skipped, **do not** rely on ClientSpace “Apply Plan Groups” for carrier→full plan list; Taylor must own a **configurable map** (or equivalent). **Which** carrier books apply follows **rebanding** (band present for a carrier → include that carrier’s plan universe). See [2.2.5](./2.2.5-Adapter-mapping-layer-to-manage-Prism-schema-dependencies.md), [2.4.3](./2.4.3-Data-normalization-prior-to-workbook-population.md).
- **Regional / zip rules:** Validate using the **annual zip file** plus a separate **state→plan** mapping (Baby Taylor has this today); combine **employer HQ** rules (e.g., tri-state) with employee zip coverage.
- **Deny list:** Maintain a **deny list** for **test**, **GAP**, **grandfathered/removed** plans (Baby Taylor uses an array today); Prism may not expose “not renewable” cleanly before 1/1 while payroll is still active.
- **Dental/vision pairing:** Enforce carrier pairing rules (e.g., MetLife dental ↔ MetLife vision) on generation and/or submission validation.
- **Sponsorship / scheduling hints:** **Plan ID naming conventions** distinguish client-sponsored vs Questco-sponsored plans; Questco analyst team to provide a key for scheduling rules (e.g., CSP ~90-day vs January 1 batch).
- **Termination date in Prism (open):** Group benefit plan **termination date** may exist in Prism; validate with **LaWanda/analysts** before relying on it—historical issues with deductions reported.
