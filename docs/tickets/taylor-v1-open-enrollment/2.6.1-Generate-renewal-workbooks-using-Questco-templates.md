# 2.6.1 - Generate renewal workbooks using Questco templates

## Traceability
- **Asana task ID:** `1213863006807796`
- **Section:** Backlog

## Product intent

Automatically produce renewal workbooks from Questco-standard Excel templates so benefits analysts do not manually copy base structure each renewal.

**Context (legacy vs. Taylor):** Today **ClientSpace** can generate a **base renewal workbook** from the batch; **BabyTaylor** (macros + automation) then **post-processes** that file—e.g. Prism API calls, census injection, checks against **carrier rate books** and **zip/regional** files. Taylor’s generation step should preserve the **Questco template contract** whether the upstream file originates from ClientSpace export or is produced entirely by Taylor.

## Product requirements (verifiable)

- System generates a workbook file from approved Questco renewal template(s) for a given client/renewal context.
- Generated file preserves required sheet names, structure, and protection expectations defined for the template version in use.
- Template version in use is identifiable (metadata, filename convention, or log) for audit.

## Success criteria

- [ ] Generation runs without manual Excel steps for a representative renewal scenario.
- [ ] Output opens in Excel; structural checklist against template spec passes (**checklist owner: Questco**).
- [ ] **Template version** is identifiable in output or logs (metadata cell/hidden sheet, file property, filename convention, or pipeline log) for audit.
- [ ] *Refine with product any remaining template inventory gaps after Questco discovery.*
