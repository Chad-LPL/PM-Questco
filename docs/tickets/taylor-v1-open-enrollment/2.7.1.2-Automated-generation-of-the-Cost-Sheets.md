## Product intent

Generated from an executed client workbook, extracting data into templatized document formats. This supports Taylor V1 open enrollment orchestration. Benefits document package generation and e-signature workflow hooks.

### Content scope (April 2026 — Anna)

The **cost sheet** is **not** identical to the CBE or benefit guide: it shows what **costs the employee per paycheck** for **elected** medical (and similar) benefits. **Employer-paid** benefits that cost the employee **nothing** belong in the **benefit guide**, not the cost sheet ([2.7.1.3](./2.7.1.3-Automated-generation-of-the-Benefit-Guide.md)).

### As-is vs target

- **Today:** Cost sheets are created **after** DocuSign on the CBE in the current process.
- **Target:** Generate **with** CBE and benefit guide for combined review ([2.7.1.4](./2.7.1.4-Automated-generation-of-the-combined-packed.md)).

### April 9, 2026 — per class, macros, pay frequency

- **One cost sheet per benefit class** so employees only see **their** rates (classes do not see each other’s employer contribution differences).
- **Output:** Generate the **cost sheet tab** as the deliverable—avoid requiring clients to run **macros** on Taylor outputs where possible ([2.6.3](./2.6.3-Backend-rewrite-of-workbook-setup-macro-logic.md)).
- **Math:** **Column Q** carries **monthly** employee contribution in the workbook flow discussed; **pay frequency** comes from **Prism** / client master—derive **per-pay** amounts from monthly + pay periods.
