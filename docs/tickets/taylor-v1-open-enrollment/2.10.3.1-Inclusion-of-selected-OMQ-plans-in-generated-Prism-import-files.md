# 2.10.3.1 - Inclusion of selected OMQ plans in generated Prism import files

## Traceability
- **Asana task ID:** `1213863006807860`
- **Section:** Backlog

## Product intent

When a client **selects** off-market quote (OMQ) plans, generated **Prism import files** (and related Taylor outputs) must **include** those plans and rates so analysts are not re-keying from scratch where automation can cover it. Ties to [2.5.4](./2.5.4-OMQ-plan-data-entry-and-management.md) for how OMQ data is captured and managed upstream.

### Context from discovery (April 2026)

- **Today:** New OMQ plans often require **manual PRISM entry** by analysts; existing-plan renewals can use **import files** (e.g. BabyTaylor-modified) per Jagger. Taylor should **close the gap** for selected OMQ plans in generated imports.
- **Workbook / packet:** Product owner confirmed **single workbook** with OMQ beside renewals ([2.10.2.1](./2.10.2.1-Injection-of-OMQ-plans-into-renewal-workbook-alongside-Prism-sourced-plans.md)); **V1** uses a **Questco Excel template** for keyed quote data ([2.5.4](./2.5.4-OMQ-plan-data-entry-and-management.md)). Imports must reflect **client-selected** OMQ plans when elections are finalized.

**April 8, 2026:** Once OMQ rows are in the workbook and the client selects plans, **downstream Prism import generation** should mirror the **master-plan** path—**Anna:** process stays the same after OMQ is injected. Partial/age-banded data may still need **manual analyst follow-up**—track in gap analysis.

## Product requirements (verifiable)

- Selected OMQ plan identifiers and rate/contribution data flow from [2.5.4](./2.5.4-OMQ-plan-data-entry-and-management.md) (or authoritative source) into **generated Prism import payloads** for the agreed file types.
- **Gap analysis:** Document what still requires manual PRISM work vs. automated for V1.
