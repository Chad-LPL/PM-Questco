# 2.7.1.4 - Automated generation of the combined packed

## Traceability
- **Asana task ID:** `1213863006807820`
- **Section:** Backlog

## Product intent

Generated from an executed client workbook, extracting data into templatized document formats. The **combined packet** includes the **CBE**, **Cost Sheet(s)**, and **Benefit Guide**. This supports Taylor V1 open enrollment orchestration, benefits document package generation, and e-signature workflow hooks.

**Generation strategy (Questco, April 2026):** With **full automation**, **CBE, cost sheets, and benefit guide can be generated together (in parallel)** because regeneration is inexpensive—unlike today’s manual process, where generating everything before **client sign-off** wasted rework. **Business rule unchanged:** the client must still **approve** what is communicated (sign the CBE / agreed elections); operations may still **send** documents per policy—separate **technical generation** from **release to client**.

**Anna (April 7, 2026):** Today the three materials are **separate steps**; **Baby Taylor** does not build the CBE. After DocuSign, cost sheet is manual and benefit guide is **outsourced** with a **multi-day** turnaround. The target is **one client-facing set** at review: CBE language can note that the client is also approving the **cost sheet** and **benefit guide** in the same bundle. Operations may not use the word **“packet”** internally—**combined deliverable** is fine. Regenerating **all three** together when something changes adds **accountability** and reduces back-and-forth.

**April 9, 2026:** Stakeholders want **CBE, cost sheet, and benefit guide** available together for the client and DocuSign, but **not** as a single sloppy **merged PDF** or **zip** that could be forwarded incorrectly—prefer **separate files** / attachments for the same signing event. See PRD [§5.8](../../prd/Taylor-v1-Open-Enrollment-Orchestration-PRD.md#58-executed-workbook--documents--signature) and [2.7.2](./2.7.2-DocuSign-Workflow-via-ClientSpace.md). **OMQ broker liability** may require **excluding** some plan pages from Questco-branded outputs—coordinate with [2.7.1.3](./2.7.1.3-Automated-generation-of-the-Benefit-Guide.md).
