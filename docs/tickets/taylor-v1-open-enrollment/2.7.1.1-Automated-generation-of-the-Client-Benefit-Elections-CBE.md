# 2.7.1.1 - Automated generation of the Client Benefit Elections (CBE)

## Traceability
- **Asana task ID:** `1213863006807814`
- **Section:** Backlog

## Product intent

Generated from an executed client workbook, extracting data into templatized document formats. This supports Taylor V1 open enrollment orchestration. Benefits document package generation and e-signature workflow hooks.

### As-is vs target (April 2026 — Anna)

- **Today:** ClientSpace generates the **CBE**; it is sent for **DocuSign**; **after** signature, staff **manually** create the **cost sheet** and rely on a **third-party** for the **benefit guide** (~**five-day** turnaround)—so the three are **not** delivered together.
- **Target:** Generate **CBE, cost sheet, and benefit guide at the same time** for client **review** so the client sees one coherent set of materials. **CBE copy** can state that signing also approves the **cost sheet** and **benefit guide** included in the deliverable ([2.7.1.4](./2.7.1.4-Automated-generation-of-the-combined-packed.md)). If elections change, **regenerate all three** to reduce churn and vendor round-trips.

### April 9, 2026 — CBE content details

- **Fees / commission / disclosures:** Updated **annually** (blanket Questco); not per-client except where template dictates.
- **Signature-page notes:** Short call-out text—today from a **ClientSpace** field; stakeholders want optional **workbook-captured notes** (e.g., on total cost / summary) with Taylor **writing back** to ClientSpace for ongoing reference.
- **Renewal plan rows:** Pulled from workbook selections; **classes** appear in the benefit-groups column; ordering typically **cheapest → most expensive** within sections (match legacy/ClientSpace behavior where feasible).
- **Voluntary block** at bottom: sourced from **workbook** (Taylor avoids Baby Taylor’s ClientSpace round-trip for that section).
